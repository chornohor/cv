# Designer CV

Static portfolio/CV website built with Eleventy, Nunjucks, SCSS, and small vanilla JavaScript modules.

## Requirements

- Node.js
- npm

## Install

```bash
npm install
```

## Local Development

```bash
npm run dev
```

This starts Eleventy with a local server and watches template/style changes.

Default local URL:

```text
http://localhost:8080/
```

## Build

```bash
npm run build
```

The production-ready static files are generated into:

```text
_site/
```

This folder is the final website output. The files inside `_site/` can be deployed to static hosting such as GitHub Pages, Netlify, Firebase Hosting, or similar services.

## Format

```bash
npm run format
```

Formats Nunjucks, SCSS, JSON, and root JS/JSON files with Prettier.

## Project Structure

```text
src/
  _data/          Global Eleventy data
  _includes/      Layouts and reusable Nunjucks components
  assets/         Images, icons, fonts, and static assets
  scripts/        Vanilla JavaScript modules
  styles/         SCSS source files
  *.njk           Site pages

_site/            Generated static website
```

## Editing Notes

- Edit source files inside `src/`.
- Do not edit files inside `_site/` manually, because they are regenerated by Eleventy.
- Shared markup should live in `src/_includes/`.
- Page-specific styles can live in `src/styles/pages/`.
- Reusable section/component styles can live in `src/styles/pages-blocks/` or `src/styles/components/`, depending on how widely they are reused.
- JavaScript behavior is split into small files in `src/scripts/` and loaded from the base layout.

## SEO Notes

The base layout includes common SEO meta tags, Open Graph tags, Twitter card tags, canonical URL support, and favicon setup.

Before deploying to a real domain, update the site URL in:

```text
src/_data/site.json
```

Set the final production domain in the `url` field so canonical and social preview URLs are correct.

## Deployment Notes

For static hosting, deploy the generated `_site/` directory.

The repository includes `.gitignore`, so generated files, dependencies, caches, logs, and local environment files are not committed.
